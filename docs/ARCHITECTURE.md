## 核心创新点：传统 vs AI + MCP

```

            ┌──────────────────────┐            ┌──────────────────────────┐
            │                      │            │                          │
```

## 六大核心模块详解

### 模块一：ApiPost MCP → 接口真相源

```
┌──────────────────────────────────────────────────────┐
│                  ApiPost / Apifox                     │
│                                                      │
│   ┌──────────┐  ┌──────────┐  ┌──────────────────┐  │
│   │ 接口文档  │  │ 环境配置  │  │ 已保存的请求      │  │
│   │ (OpenAPI) │  │ (多环境)  │  │ (预执行脚本/断言) │  │
│   └─────┬────┘  └─────┬────┘  └────────┬─────────┘  │
│         │             │                │             │
│         └─────────────┴────────────────┘             │
│                       │                              │
│              ┌────────▼────────┐                     │
│              │   MCP Server    │                     │
│              │                 │                     │
│              │ 暴露的能力:      │                     │
│              │ · get_api_list  │ ← 获取所有接口列表   │
│              │ · get_api_detail│ ← 获取接口详情       │
│              │ · send_request  │ ← 直接调试接口       │
│              │ · get_schemas   │ ← 获取数据模型       │
│              │ · get_env_vars  │ ← 获取环境变量       │
│              └────────┬────────┘                     │
└───────────────────────┼──────────────────────────────┘
                        │ MCP 协议
                        ▼
              AI Agent 可直接:
              · "读取所有CRM登录相关接口"
              · "调用登录接口测试一下"
              · "对比测试环境和生产环境差异"
```

**价值**：接口文档即测试用例的数据源，无需手动维护 Excel，接口变更时 MCP 实时感知。

### 模块二：AI 用例智能生成引擎

```
输入: ApiPost MCP 提供的接口元数据
      ┌─────────────────────────────────┐
      │ POST /api/crm/customer/login   │
      │ Body: { account, password,     │
      │   identity_type, verify_code } │
      │ Response: { code, msg, data }  │
      └──────────────┬──────────────────┘
                     │
          ┌──────────▼──────────┐
          │    LLM (Claude/GPT)  │
          │                      │
          │  分析接口语义:        │
          │  · 哪些是必填字段?    │
          │  · 哪些字段有格式要求? │
          │  · 业务规则是什么?    │
          │  · 哪些边界值需测试?  │
          └──────────┬──────────┘
                     │
          ┌──────────▼──────────────────────────────────┐
          │             输出: pytest 测试代码             │
          │                                              │
          │  # 正向用例: 正确账号密码登录                  │
          │  def test_login_success():                   │
          │      ...                                     │
          │                                              │
          │  # 逆向用例: AI 自动推演的异常场景             │
```

```python
assert response["data"]["token"] is not None

@ai_assert("登录成功后应返回有效token和用户信息")
def test_login(response):
    # AI 自动验证:

    # 2. data.token 格式合法 (JWT结构)
    # 3. data.user_info 包含必要字段
    # 4. 即使字段名变了(如token→access_token), AI也能语义匹配
    pass

# AI Schema 断言: 对比接口文档与实际响应
@schema_assert(api_name="crm_login")
def test_login_schema(response):
    # 自动从 ApiPost MCP 获取接口定义
    # 对比实际响应与文档定义是否一致
    # 检测: 字段增删、类型变更、必填项变化
    pass
```

### 模块四：Self-Healing 自愈引擎

```
流程:
  
  接口变更                用例失败                AI 分析               自动修复
  ┌─────┐              ┌─────────┐           ┌──────────┐          ┌──────────┐
  │开发改│   →→→→→→→   │ pytest   │  →→→→→→  │ LLM 读取 │  →→→→→→  │ 自动更新  │
  │了接口│              │ 报错     │           │ 报错信息 │          │ 测试代码  │
  └─────┘              └─────────┘           │ + 接口文档│          └──────────┘
                                             └──────────┘
  
  示例:
  开发把 login 接口的 response.data.token 改成了 response.data.access_token
  
  传统: 用例全红, 测试同学逐个排查修复
  AI:  识别到"字段名变更" → 建议替换 token→access_token → 自动/确认后修复
```

## 技术选型建议

| 层级 | 技术栈 | 说明 |
|------|--------|------|
| **AI 模型** | Claude 3.5 Sonnet / GPT-4o | 代码生成能力强，支持 Function Calling |
| **MCP 框架** | Anthropic MCP SDK (Python) | 官方 SDK，标准化协议 |
| **接口管理** | ApiPost / Apifox | 已支持 MCP Server，提供接口元数据 |
| **测试框架** | pytest | 生态丰富，插件化，AI 集成友好 |
| **报告** | Allure + pytest-testreport | 趋势分析 + 实时报告 |
| **CI/CD** | Jenkins / GitLab CI | 定时巡检 + 发布触发 |
| **通知** | 飞书/钉钉 Webhook | 实时推送测试结果 |
| **数据库** | pymysql → MCP 封装 | AI 可直接查询校验 |
